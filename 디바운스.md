지금 네 구조를 최대한 유지하면서 **진짜 debounce**를 넣는다면 이렇게 가는 게 가장 깔끔해.

핵심은:

```text
새 입력 발생
→ 이전 저장 task 취소
→ 새 task 예약
→ 700ms 동안 추가 입력 없으면 저장
```

---

먼저 `save_task`를 하나 만들어야 해.

```rust
use tokio::task::JoinHandle;

let mut save_task = use_signal(|| None::<JoinHandle<()>>);
```

그리고 `use_auto_save`에 넘겨.

```rust
use_auto_save(
    text_value,
    original_content,
    current_note_id,
    list_resource,
    save_task,
);
```

---

그리고 autosave:

```rust
pub fn use_auto_save(
    text_value: Signal<String>,
    original_content: Signal<String>,
    current_note_id: Signal<Option<i64>>,
    mut list_resource: Resource<Vec<NoteSummary>>,
    mut save_task: Signal<Option<tokio::task::JoinHandle<()>>>,
) {
    let pool = use_context::<sqlx::PgPool>();

    use_effect(move || {
        let current_text = text_value.read().clone();
        let old_text = original_content.peek().clone();

        if current_text.is_empty() || current_text == old_text {
            return;
        }

        // 이전 예약 취소
        if let Some(task) = save_task.write().take() {
            task.abort();
        }

        let pool_cloned = pool.clone();

        let mut id_state = current_note_id;
        let mut original_state = original_content;

        let current_id = *id_state.read();

        let handle = tokio::spawn(async move {
            tokio::time::sleep(
                std::time::Duration::from_millis(700)
            ).await;

            let saved = match current_id {
                Some(id) => {
                    update_data(id, &current_text, pool_cloned)
                        .await
                        .is_ok()
                }

                None => {
                    match insert_data(&current_text, pool_cloned).await {
                        Ok(new_id) => {
                            id_state.set(Some(new_id));
                            true
                        }
                        Err(_) => false,
                    }
                }
            };

            if saved {
                original_state.set(current_text);
                list_resource.restart();
            }
        });

        save_task.set(Some(handle));
    });
}
```

---

하지만 여기서 하나 더.

Dioxus에서 `JoinHandle<()>`이 `Signal`에 들어갈 수 있는지는 버전에 따라 제약이 있을 수 있어.

그럴 경우 더 안전한 방법은:

```rust
use_signal(|| 0u64)
```

같은 generation counter를 쓰거나,

```rust
use_coroutine
```

으로 debounce 전용 루프를 만드는 거야.

---

그리고 네 코드를 보면서 한 가지 더 눈에 띄는 게 있어.

```rust
let current_id = *id_state.read();
```

를 spawn 전에 읽고 있잖아?

이러면:

```text
새 노트 생성
→ id 없음(None)
→ debounce 시작

500ms 후
→ 사용자가 다른 노트 클릭

700ms 후
→ 여전히 None 기준으로 insert 실행
```

같은 race가 생길 수 있어.

그래서 실제로는 저장 직전에 다시 읽는 구조가 더 안전할 수도 있어.

---

현재 단계에서는:

1. `use_effect` 복구
2. `original_content.peek()`
3. `JoinHandle.abort()` debounce
4. 저장 성공 시 `restart()`

이 4개만 적용해도 지금 겪는 "무한 저장/과도한 저장" 문제는 거의 해결될 거야.
